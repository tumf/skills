# 共通ヒューリスティクス

## 不要テストのシグナル

- 対応する production code / export / route / command が消えている
- 現在の docs や README で説明されていない旧仕様を固定している
- 旧関数名・旧エラーメッセージ・旧レスポンス形だけを検証している
- `skip` / `todo` / `deprecated` / `legacy` が長期間残っている
- より上位のテストが同じ振る舞いを十分カバーしている

## 重複テストのシグナル

- セットアップがほぼ同じ
- 入力バリエーションが意味的に薄い
- 期待値が同じ
- 守っている振る舞いが同じ
- helper 抽出や parameterized test で圧縮できる

## flaky のシグナル

- sleep に依存している
- 現在時刻に依存している
- random / UUID を固定していない
- 実行順や shared state に依存している
- 環境変数やファイルシステム状態を引きずる
- ネットワークや外部サービスに依存している

## 推奨分類ラベル

- 現役
- 要確認
- 削除候補
- 重複候補
- 統合候補
- hidden integration
- flaky 候補
