# Framework Selection Playbooks

Use this file when choosing which brainstorming method to apply.

## Fast selection table

### Need explosive divergence
Use:
- SCAMPER
- Random stimulus

### Need a solo thinking tool
Use:
- Mind map
- Why / How ladder

### Need a meeting structure
Use:
- Six Thinking Hats
- Brainwriting

### Need synthesis after idea generation
Use:
- KJ method

### Need business framing
Use:
- JTBD
- ERRC

### Need validation path
Use:
- Design Sprint

## Recommended combinations

### 1. Raw expansion -> structure
- SCAMPER -> KJ method
- Random stimulus -> KJ method

Use when:
- the user needs many options first, then synthesis

### 2. Exploration -> business grounding
- Mind map -> JTBD
- Random stimulus -> JTBD

Use when:
- ideas are creative but not yet tied to user value

### 3. Discussion -> decision
- Six Thinking Hats -> ERRC
- Brainwriting -> ERRC

Use when:
- the team has many opinions and needs strategic narrowing

### 4. Strategy -> execution
- JTBD -> Why / How ladder
- ERRC -> Design Sprint

Use when:
- the user wants to move from framing into action

## Practical routing rules

If the user says:
- "ideas are stuck" -> start with SCAMPER or Random stimulus
- "I want to think alone" -> start with Mind map
- "we need a workshop" -> start with Six Thinking Hats or Brainwriting
- "we already have too many ideas" -> use KJ method
- "make this useful for the business" -> use JTBD or ERRC
- "how do we test this quickly" -> use Design Sprint

## Output guidance

When using a framework:
- name the chosen framework explicitly
- say why it fits this task
- group the ideas by the framework's logic
- do not dump all frameworks unless the user explicitly asks for a comparison

## Escalation pattern

If the first framework does not produce useful output:
1. Switch to a more divergent framework if ideas are too narrow.
2. Switch to a more structural framework if ideas are too messy.
3. Switch to a business framework if ideas are interesting but weak in value.

## AI-agent automation outline

If asked to automate brainstorming with AI agents, use this pipeline:
1. Theme clarification agent
2. Framework selector agent
3. Divergence agent
4. Clustering agent
5. Ranking or business-framing agent
6. Experiment-design agent

Keep the system modular so the framework can change without redesigning the whole pipeline.
